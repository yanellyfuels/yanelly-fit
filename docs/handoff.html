Yanelly Fuels — Handoff Guide

Passing the website over to Yanelly, and building it with ChatGPT Codex.

This walks through giving Yanelly full ownership of the website so she controls it, keeps it online for free, and can keep improving it with ChatGPT Codex — while Jon stays on as a helper (without owning it). The whole thing takes about 15 minutes and needs no coding.

What Yanelly is getting

  • A live website — currently at brojonbeast.github.io/yanelly-fit
  • All the source files, kept in a GitHub repository ("repo")
  • Automatic publishing & free hosting — any approved change goes live in about a minute
  • A built-in email-capture funnel (the free Balanced Plate Cheat Sheet) ready to connect to Kit

Before you start — quick checklist

  • Yanelly has a free GitHub account (github.com → Sign up) — note her username.
  • Yanelly has a ChatGPT plan that includes Codex (Plus, Pro, or Business).
  • Jon knows his GitHub username (brojonbeast) so he can be re-added afterward.

AJon transfers ownership to Yanelly  ~5 min

On Jon's computer, signed in to GitHub:

  1. Open the repo: github.com/brojonbeast/yanelly-fit
  2. Click Settings (top of the repo), then scroll to the bottom to the red "Danger Zone."
  3. Click Transfer.
  4. Type the repo name to confirm, then enter Yanelly's GitHub username as the new owner.
  5. Confirm. GitHub emails Yanelly a request to accept the transfer.
Nothing is lost. All the files, history, and settings move with the repo — it's the same project, just under Yanelly's account.

BYanelly accepts & switches the site back on  ~5 min

  1. Yanelly opens the email (or her GitHub notifications) and clicks Accept. The repo now lives at github.com/<her-username>/yanelly-fit.
  2. Make sure it's Public: Settings → General → (bottom) Change visibility → Public. Required for free hosting.
  3. Turn the website on: Settings → Pages → Build and deployment → Source = "GitHub Actions."
  4. If prompted, allow Actions: Settings → Actions → General → Allow all actions.
  5. Wait 1–2 minutes. The site is now live at https://<her-username>.github.io/yanelly-fit/
Heads up — the web address changes. It moves from brojonbeast.github.io/… to Yanelly's username. Update the link in her Instagram bio to the new address. (Later, a custom domain like yanellyfuels.com would make the address permanent so this never happens again.)

CKeep Jon's access  ~2 min

So Jon can still help without owning it — on Yanelly's repo:

  1. Settings → Collaborators (and teams) → Add people.
  2. Enter brojonbeast and choose a role:
    • Admin — full help, including settings (recommended).
    • Write — can edit files and merge changes.
  3. Jon accepts the email invite. Done — he can jump in anytime.

DConnect ChatGPT Codex to the repo  ~5 min

  1. In ChatGPT, open Codex (the coding tool in the sidebar).
  2. Connect GitHub — authorize the ChatGPT/Codex GitHub app for her account.
  3. Grant it access to the yanelly-fit repo (you can limit it to just this one).
  4. Select yanelly-fit as the working repo; Codex sets up its environment.
  5. Give it a task (see examples below). Codex works on a copy and opens a Pull Request — a proposed change — on GitHub for you to approve.
Button names inside ChatGPT may shift over time; if something looks different, search OpenAI's Codex help. Codex requires a ChatGPT plan that includes it.

★How building works day-to-day

The flow is always the same:

Ask Codex → it opens a Pull Request → you review it on GitHub → click Merge → the site updates itself in ~1 minute.

The golden rule: a change only goes live once it's merged into the "main" branch. Until then, the live site is untouched — so you can experiment safely.

You can also make quick text edits without Codex: on GitHub, open a file → click the pencil icon → edit → Commit changes.

Things you can ask Codex

  • "Add a recipe card for protein pancakes."
  • "Change the hero headline to '…'."
  • "Add an FAQ: 'Do you offer meal plans?'"
  • "Make the strawberry accent a deeper berry."
  • "Add a new free guide called 'Meal Prep 101'."
  • "Update my About section with this text: …"

Tip: ask for one change at a time, and have Codex describe or preview it before you merge.

→Next win: turn on email capture (Kit)

The site already has the opt-in forms and the free cheat sheet built in — they just need to be connected to an email tool so signups are actually saved. Recommended: Kit (free up to 10,000 subscribers, built for creators).

  1. Yanelly creates her own free Kit account at kit.com.
  2. In Kit, make a form and set it to automatically deliver the cheat-sheet PDF by email.
  3. Ask Codex: "wire the opt-in forms on my site to my Kit account" (she'll paste her Kit form details).

After that, every signup becomes a subscriber and gets the cheat sheet automatically — the foundation for newsletters, digital products, and 1:1 coaching later.

Mini-glossary

  • Repository (repo) — the folder of all the site's files + history.
  • Branch — a safe copy to make changes on.
  • Pull Request (PR) — a proposed change you review before it's live.
  • Merge — approving a PR so it becomes part of the live site.
  • GitHub Pages — the free hosting that serves the website.
  • main — the live version of the site.

Need help?

Jon stays on as a collaborator and can step in anytime. If anything looks broken, describe what you see (a screenshot helps) and either Codex or Jon can fix it. Nothing you do in a Pull Request can break the live site until it's merged — so explore freely.
